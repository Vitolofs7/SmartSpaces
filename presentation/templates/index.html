Key Features

  • 📊 Real-time Availability: Check available spaces for any date and time range
  • 🔄 Smart Bookings: Create, modify, and cancel reservations with automatic conflict detection
  • 👤 User Management: Complete user lifecycle with activation/deactivation controls
  • 🏢 Space Types: Support for generic spaces and specialized meeting rooms with equipment tracking
  • 📱 REST API: 19 fully documented endpoints for integration with external systems
  • 🌐 Web Interface: Modern, responsive HTML interface for direct interaction
  • 📊 Observability: Comprehensive logging and error handling with user-friendly pages
  • 🔒 Data Integrity: SQLite-backed persistence with transactional safety

System Architecture

SmartSpaces is built on a layered architecture with clear separation of concerns:

  • Domain Layer: Business logic and entities (User, Space, Booking)
  • Application Layer: Use case orchestration (Services)
  • Infrastructure Layer: Data persistence (SQLite repositories)
  • Presentation Layer: Web API (Flask) and console menu interface

Getting Started

For API consumers: Use any HTTP client to interact with the endpoints listed in the API Help page. The system supports both JSON responses (for API integrations) and HTML rendering (for web browsing).

For system administrators: The same SQLite database is accessible from both the REST API (this interface) and the console menu application, allowing seamless data sharing.
