Common Use Cases

🔍 Retrieve Information (GET requests)

  • GET /users - List all users
  • GET /users/<user_id> - Get a specific user
  • GET /spaces - List all spaces
  • GET /spaces/<space_id> - Get a specific space
  • GET /bookings - List all bookings
  • GET /bookings/<booking_id> - Get a specific booking
  • GET /spaces/disponibles/2026-05-15T09:00:00/2026-05-15T17:00:00 - Find available spaces in a time range

➕ Create Resources (POST requests)

  • POST /users/nuevo/<id>/<name>/<surname1>/<surname2> - Create a new user
  • POST /spaces/nuevo/<name>/<capacity>/<type> - Create a new space
  • POST /spaces/nueva-sala/<...> - Create a new meeting room
  • POST /bookings/nueva/<user>/<space>/<start>/<end> - Create a new booking

🔄 Modify Resources (POST requests)

  • POST /bookings/<id>/cancelar - Cancel a booking
  • POST /bookings/<id>/finalizar - Mark booking as finished
  • POST /bookings/<id>/reprogramar/<new_start>/<new_end> - Reschedule a booking
  • POST /users/<id>/desactivar - Deactivate a user

Date Format

When specifying dates and times (for bookings, availability searches, etc.), use ISO 8601 format:

YYYY-MM-DDTHH:MM:SS

Example: 2026-05-15T14:30:00

HTTP Status Codes

Code Meaning Example
200 OK - Request succeeded GET request returned data
302 Redirect - POST followed by GET After creating a resource
400 Bad Request - Invalid data Malformed date, invalid parameters
404 Not Found - Resource doesn't exist User ID not in database
409 Conflict - Duplicate or state conflict Creating a user with duplicate ID
500 Server Error - Internal problem Unexpected server error
